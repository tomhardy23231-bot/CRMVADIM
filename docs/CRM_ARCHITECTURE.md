# Документация и Архитектура CRM Системы

Наша CRM — это современное высокопроизводительное веб-приложение, разработанное на стеке Next.js + React + Prisma (база данных SQLite). CRM предназначена для ведения мебельных (и других) тендеров с фокусом на глубокий финансовый анализ План-Факт.

---

## 1. Общая Архитектура

### Технологический стек
* **Frontend:** React 18, Next.js 14 (App Router), TailwindCSS, компоненты ShadCN UI (Radix UI). Дополнительно: `recharts` для аналитики, `xlsx` для выгрузки отчётов.
* **Backend:** Next.js API Routes (встроенный Node.js сервер).
* **Database:** PostgreSQL (хостинг Neon) через ORM Prisma (`prisma/schema.prisma`).
* **Анимации:** Framer Motion (скелетоны и UX).
* **Интеграция:** HTTP-запросы REST к опубликованному API 1С:Предприятие.

### Основные директории
* `src/app`: Маршрутизация, страницы (Dashboard) и серверные API-эндпоинты (`/api/orders`, `/api/1c/...`).
* `src/components`: Визуальные компоненты.
  * `crm/tabs`: Компоненты для работы со вкладками внутри тендера (Бюджет, Спецификация, Платежи 1С и т.д.).
* `src/lib`: Системная логика, глобальный контекст (`crm-context.tsx`), утилиты, типы TypeScript.
* `prisma`: Схема базы данных (`schema.prisma`).

## 2. Модель Данных (База Данных)

Ключевой сущностью является **Order** (Заказ/Тендер).
С заказом связаны другие таблицы в формате "один ко многим":

1. **SpecItem (Спецификация)**: Список материалов, работ и фурнитуры. Сюда импортируется информация из документов "Реализация" (Себестоимость и Сумма продажи из 1С).
2. **BudgetItem (Статья Бюджета)**: Разделы финансового плана. Каждая статья имеет `plan` (ожидание) и `fact` (реальность, подтягивается автоматически из платежей или пишется вручную).
   * **Tranche (Транши)**: Дочерняя таблица к статьям бюджета для точной разбивки плановой суммы по датам. Раньше использовалось планирование по неделям, теперь используется точная ISO-дата (`plannedDate`). Если статья не разбита на несколько траншей вручную (например, 10 000 ₴ за логистику одним платежом), система автоматически создает "скрытый" транш, позволяя выбирать дату прямо в строке статьи бюджета.
3. **Payment1C (Платежи из 1С)**: Сырые транзакции, импортируемые из 1С. Каждая транзакция хранит:
   * Дату, Документ (ПКО, РКО, выписка банка)
   * Приход (Income) / Расход (Expense)
   * Статью ДДС (Текстовое значение из 1С).
   * **Связь с бюджетом:** `budgetItemId` (ID статьи, к которой привязан этот факт оплат).

## 3. Логика работы (Business Flow)

### 3.1. Создание и Импорт Тендера
1. Менеджер нажимает "Данные из 1С" -> "Импорт из 1С".
2. Открывается диалог. CRM отправляет запрос `GET /api/1c/contractors` на бэкенд, который маршрутизируется в 1С. 1С возвращает список контрагентов (первые 100).
3. Менеджер выбирает клиента (например, "5 СТОЛОВ") и нажимает Импорт.
4. CRM делает детальный запрос `GET /api/1c/contractor/search?name=5 СТОЛОВ`. 1С возвращает:
   * Массив продаж (`sales`)
   * Массив расчетов (`settlements`).

### 3.2. Авто-Синхронизация Платежей (Умный План-Факт)
При импорте бэкенд CRM делает сложную работу:
1. CRM проверяет, пришла ли от 1С "Статья ДДС" вместе с платежом (в поле `article`).
2. В цикле создания платежа в базе данных (см. файл `src/app/api/orders/route.ts`), бэкенд ищет совпадение по имени: `if (article.toLowerCase() == budgetItem.name.toLowerCase())`.
3. Если Имя статьи бюджета полностью совпадает с текстом статьи из 1С, платеж **автоматически** привязывается к этой статье.
4. **Результат:** На вкладке Бюджетирования колонка "Факт из 1С" автоматически суммирует все платежи по этой привязанной статье, и считается отклонение "План-Факт".

> Если статья не совпала или пришла пустой, платеж просто ложится в раздел "Оплаты План-Факт", где менеджер может вручную выбрать выпадающий список и прикрепить транзакцию к нужной статье бюджета.

### 3.3. Разделение Прибыли и Затрат
CRM четко разграничивает маржинальность и имеет гибридную систему расчета:
1. **Плановая сумма заказа (Выручка):** При импорте из 1С, она высчитывается автоматически (максимум из заказов и продаж). Однако менеджер может нажать ✏️ и перевести сумму в **Ручной ввод** (`isAmountManual`). В этом режиме CRM защищает цифру от последующих перезаписей 1С. В интерфейсе ручной ввод подсвечивается зеленым цветом.
2. **Себестоимость Спецификации (Материалы):** При импорте формируется неприкасаемая статья бюджета "Материалы (План)". Ее Факт жестко привязан к Плану (оба равны `plannedCost` из 1С). Менеджерам запрещено (и не нужно) привязывать сырые платежки за доски или гвозди к этой статье — она является постоянной величиной расходов.
3. **Фактическая маржа (True Profit):** Рассчитывается динамически с использованием **"Гибридной себестоимости" (Smart Cost)**. Алгоритм анализирует каждую статью расходов (кроме материалов, которые фиксированы из 1С). Если статье бюджета привязан реальный платеж (Факт) — берется он. Если платежей еще не было — в качестве ожидания берется План. От Суммы Заказа отнимается эта гибридная себестоимость для отображения реальной маржи.

### 3.4. Идемпотентность (Повторный Импорт)
Система спроектирована так, чтобы можно было нажимать "Импорт из 1С" каждый день.
При повторном импорте того же контрагента (УНН или ID 1С совпадает), CRM **не дублирует тендер**. Вместо этого она:
* Ищет транзакции, которых еще нет в базе.
* Дописывает только новые платежи (строит Set из существующих номеров документов).
* Сохраняет ваши ручные настройки и связи.

## 4. Пользовательский Интерфейс (UI)
* **Дашборд:** Список карточек с тендерами, быстрые переключатели 9-ти статусов (Проектирование -> Закупка -> ... -> Выполнен -> Оплачен).
* **Внутренняя карточка заказа:** Содержит интерактивный таймлайн и "Гибридную себестоимость" проекта онлайн.
* **Бюджетирование:** Редактируемая таблица (Google Sheets-подобная), где менеджеры планируют расходы по траншам или ставят точные даты оплаты прямо в общих строках.
* **Платёжный календарь (Фин. Контроль):** Глобальный премиальный Banking-интерфейс, агрегирующий все транши/расходы и оплаты от клиентов. Показывает кассовые разрывы (красные зоны), графики Cash Flow и поддерживает сквозную детализацию прямо из таблицы.
* **Спецификация (Отображается из 1С):** Режим только для чтения, выводит номенклатуру, цены и себестоимости с подсветкой убыточных позиций.
* **Платежи 1С:** Табличный лог, показывающий все движения денег в 1С с возможностью привязки.

Эта CRM уникальна тем, что объединяет неповоротливую бухгалтерскую точность 1С с быстрой и понятной веб-оболочкой для руководителей проектов и менеджеров.
