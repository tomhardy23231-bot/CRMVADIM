# Инструкция по настройке интеграции 1С:Предприятие и CRM

Данное руководство предназначено для настройки обмена данными между рабочей (боевой) базой 1С:Предприятие (УТ, УПП, ERP) и веб-CRM системой. Обмен осуществляется через HTTP-сервис на стороне 1С.

---

## 1. Подготовка конфигурации 1С

### 1.1 Создание HTTP-сервиса
1. Откройте **Конфигуратор** рабочей базы 1С.
2. В дереве метаданных найдите ветку **Общие -> HTTP-сервисы**.
3. Добавьте новый HTTP-сервис. Назовите его `CrmApi`.
4. В поле **Корневой URL** укажите: `crm`.
5. В поле **Повторное использование сеансов** выберите: `Автоматически`.

### 1.2 Настройка шаблонов URL
Внутри созданного HTTP-сервиса `CrmApi` создайте 4 шаблона URL:

1. **Ping**
   * Шаблон: `/ping`
   * Добавьте метод `GET`. 
   * В поле "Обработчик" укажите функцию `PingGet`.

2. **Orders**
   * Шаблон: `/orders`
   * Добавьте метод `GET`. 
   * В поле "Обработчик" укажите функцию `OrdersGet`.

3. **ContractorInfo**
   * Шаблон: `/contractor`
   * Добавьте метод `GET`.
   * В поле "Обработчик" укажите функцию `ContractorGet`.

4. **ContractorsList**
   * Шаблон: `/contractors`
   * Добавьте метод `GET`.
   * В поле "Обработчик" укажите функцию `ContractorsListGet`.

## 2. Создание Общего Модуля

1. В ветке **Общие -> Общие модули** создайте новый модуль. Назовите его `CrmApi` (чтобы совпадало с обработчиками).
2. В свойствах модуля установите галочки:
   * Сервер
   * Вызов сервера
   * Привилегированный (обязательно, чтобы обойти ограничения RLS при чтении).
3. Откройте модуль и вставьте в него **полный код, приведенный ниже**.

> **Важно:** В этом коде реализована защита паролем `x-api-key: MySecretKey123`. Вы можете изменить пароль в строке `АпиКлюч <> "MySecretKey123"`, но тогда обязательно обновите его и в CRM (файл `src/app/api/1c/contractor/search/route.ts`).

### Умный алгоритм платежей (Взаиморасчеты)
Код написан так, чтобы быть устойчивым (fault-tolerant) к специфике вашей конфигурации. Он автоматически извлекает статью ДДС (или НазначениеПлатежа / Комментарий) и оборачивает всё в `Попытка-Исключение`, чтобы в случае отсутствия какого-либо поля в документе, 1С не прерывала отдачу массива платежей.

### Полный код модуля CrmApi:

```bsl

	АпиКлюч = Запрос.Заголовки.Получить("x-api-key");

	Контрагент = СпрКонтрагенты.НайтиПоНаименованию(ИмяИлиУНН, Истина);

	Если Контрагент.Пустая() Тогда
		Ответ = Новый HTTPСервисОтвет(404);
		Ответ.УстановитьТелоИзСтроки("Контрагент не найден");

	КонецЕсли;

	СтруктураКонтрагента = Новый Структура;

	СтруктураКонтрагента.Вставить("name", Контрагент.Наименование);
	РезультатJSON.Вставить("contractor", СтруктураКонтрагента);
	
	// 1. ВЗАИМОРАСЧЕТЫ (Платежи) - ОСТАВЛЯЕМ ТОЛЬКО ДЕНЬГИ

		|	Взаиморасчеты.Период КАК Период,
		|	Взаиморасчеты.Регистратор КАК Регистратор,
		|	Взаиморасчеты.Регистратор.Представление КАК Документ,
		|	Взаиморасчеты.СуммаВзаиморасчетов КАК Сумма

		|	Взаиморасчеты.Контрагент = &Контрагент
		|УПОРЯДОЧИТЬ ПО Период УБЫВ";

	ЗапросВзаиморасчеты.УстановитьПараметр("Контрагент", Контрагент);
	
	ВзаиморасчетыМассив = Новый Массив;
	Попытка
		ВыборкаВР = ЗапросВзаиморасчеты.Выполнить().Выбрать();
		Пока ВыборкаВР.Следующий() Цикл
			
			ДокИмяДлинное = НРег(ВыборкаВР.Документ);
			Выручка = 0; Оплата = 0;
			
			// === ОПРЕДЕЛЯЕМ ДОХОД/РАСХОД (ЗАЩИТА ОТ ОШИБОК КОНФИГУРАЦИИ) ===

			 Или Найти(ДокИмяДлинное, "приходный кассовый ордер") > 0
			 Или Найти(ДокИмяДлинное, "поступление безналичных") > 0
			 Или Найти(ДокИмяДлинное, "поступление на расчетный") > 0 Тогда
				Выручка = Макс(ВыборкаВР.Сумма, -ВыборкаВР.Сумма); // Входящие деньги
			ИначеЕсли Найти(ДокИмяДлинное, "платежное поручение исходящее") > 0
			 Или Найти(ДокИмяДлинное, "расходный кассовый ордер") > 0
			 Или Найти(ДокИмяДлинное, "списание безналичных") > 0
			 Или Найти(ДокИмяДлинное, "списание с расчетного") > 0 Тогда
				Оплата = Макс(ВыборкаВР.Сумма, -ВыборкаВР.Сумма); // Исходящие деньги
			Иначе
				// Пропускаем (это производство, возврат или реализация)
				Продолжить;

				КонецЕсли;
			Исключение
			КонецПопытки;
			// ===================================
			
			Стр.Вставить("article", СокрЛП(СтатьяТекст));
			ВзаиморасчетыМассив.Добавить(Стр);
		КонецЦикла;
	Исключение
		Сообщить("CRM API: ошибка взаиморасчетов: " + ОписаниеОшибки());
	КонецПопытки;

	// 2. ПРОДАЖИ И СЕБЕСТОИМОСТЬ
	ТекстПродажи = "ВЫБРАТЬ
		|	Продажи.Период КАК Период,
		|	Продажи.Регистратор.Представление КАК Документ,
		|	Продажи.Номенклатура.Наименование КАК Товар,
		|	Продажи.Количество КАК Количество,
		|	Продажи.Стоимость КАК СуммаПродажи,
```

## 3. Публикация HTTP-сервиса на боевом сервере

1. Запустите 1С от имени Администратора (чтобы были права изменять файлы IIS / Apache).
2. Перейдите в **Администрирование -> Публикация на веб-сервере**.
3. Укажите имя публикации (например, `upp_prod`).
4. Поставьте галочку **"Публиковать HTTP-сервисы по умолчанию"**.
5. Убедитесь, что в списке сервисов стоит галочка напротив вашего созданного `CrmApi`.
6. Нажмите **"Опубликовать"**.
7. Если вы используете IIS, перезапустите пул приложений или выполните `iisreset` в командной строке. На Apache — перезапустите сервис apache2 (`systemctl restart apache2`).

## 4. Подключение CRM к боевой базе

Чтобы CRM начала брать данные из вашей новой боевой базы, нужно изменить ссылку в исходном коде:

1. Откройте файл `src/app/api/1c/contractor/search/route.ts` (и аналогично `src/app/api/1c/contractors/route.ts`).
2. Найдите строку с `fetch("http://94.130.238.102/upp2_crm_test/hs/crm/...")`.
3. Замените URL на IP-адрес вашего сервера 1С, имя публикации (`upp_prod`), и корректные учетные данные авторизации (у пользователя 1С "admin" должен быть пароль).
4. Пример: `http://ВАШ_IP/upp_prod/hs/crm/contractor?name=...`

CRM немедленно начнет получать данные из боевой базы в реальном времени.
